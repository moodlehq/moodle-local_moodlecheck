    <h1 id="firstHeading" class="firstHeading">Core APIs</h1>
    <div id="bodyContent">
        <div id="siteSub">From MoodleDocs</div>
        <div id="contentSub"></div>

</p><p>They are implemented in different ways, but they are essential when writing Moodle plugins.
</p><p><br />
</p>

<h2><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=1" title="Edit section: Most-used General APIs">edit</a>]</span> <span class="mw-headline" id="Most-used_General_APIs">Most-used General APIs</span></h2>
<p>These APIs are very critical and will be used by nearly every Moodle plugin.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=2" title="Edit section: Access API (access)">edit</a>]</span> <span class="mw-headline" id="Access_API_.28access.29"> Access API (access) </span></h3>
<p>The <a href="/dev/Access_API" title="Access API">Access API</a> gives you functions so you can determine what the current user is allow to do, and it allows modules to extend Moodle with new capabilities.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=3" title="Edit section: Data manipulation API (dml)">edit</a>]</span> <span class="mw-headline" id="Data_manipulation_API_.28dml.29"> Data manipulation API (dml) </span></h3>
<p>The <a href="/dev/Data_manipulation_API" title="Data manipulation API">Data manipulation API</a> allows you to read/write to databases in a consistent and safe way.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=4" title="Edit section: File API (files)">edit</a>]</span> <span class="mw-headline" id="File_API_.28files.29"> File API (files) </span></h3>
<p>The <a href="/dev/File_API" title="File API">File API</a> controls the storage of files in connection to various plugins.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=5" title="Edit section: Form API (form)">edit</a>]</span> <span class="mw-headline" id="Form_API_.28form.29"> Form API (form) </span></h3>
<p>The <a href="/dev/Form_API" title="Form API">Form API</a> defines and handles user data via web forms.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=6" title="Edit section: Logging API (log)">edit</a>]</span> <span class="mw-headline" id="Logging_API_.28log.29"> Logging API (log) </span></h3>
<p>The <a href="/dev/Logging_API" title="Logging API">Logging API</a> allows you to add new entries to the Moodle log and define how they get displayed in reports.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=7" title="Edit section: Navigation API (navigation)">edit</a>]</span> <span class="mw-headline" id="Navigation_API_.28navigation.29"> Navigation API (navigation) </span></h3>
<p>The <a href="/dev/Navigation_API" title="Navigation API">Navigation API</a> allows you to manipulate the navigation tree to add and remove items as you wish.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=8" title="Edit section: Page API (page)">edit</a>]</span> <span class="mw-headline" id="Page_API_.28page.29"> Page API (page) </span></h3>
<p>The <a href="/dev/Page_API" title="Page API">Page API</a> is used to set up the current page, add JavaScript, and configure how things will be displayed to the user.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=9" title="Edit section: Output API (output)">edit</a>]</span> <span class="mw-headline" id="Output_API_.28output.29"> Output API (output) </span></h3>
<p>The <a href="/dev/Output_API" title="Output API">Output API</a> is used to render the HTML for all parts of the page.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=10" title="Edit section: String API (string)">edit</a>]</span> <span class="mw-headline" id="String_API_.28string.29"> String API (string) </span></h3>
<p>The <a href="/dev/String_API" title="String API">String API</a> is how you get language text strings to use in the user interface.   It handles any language translations that might be available.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=11" title="Edit section: Upgrade API (upgrade)">edit</a>]</span> <span class="mw-headline" id="Upgrade_API_.28upgrade.29"> Upgrade API (upgrade) </span></h3>
<p>The <a href="/dev/Upgrade_API" title="Upgrade API">Upgrade API</a> is how your module installs and upgrades itself, by keeping track of it's own version.
</p><p><br />
</p>
<h2><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=12" title="Edit section: Other General APIs">edit</a>]</span> <span class="mw-headline" id="Other_General_APIs">Other General APIs</span></h2>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=13" title="Edit section: Backup API (backup)">edit</a>]</span> <span class="mw-headline" id="Backup_API_.28backup.29"> Backup API (backup) </span></h3>
<p>The <a href="/dev/Backup_API" title="Backup API">Backup API</a> defines exactly how to convert course data into XML for backup purposes, and the <a href="/dev/Restore_API" title="Restore API">Restore API</a> describes how to convert it back the other way.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=14" title="Edit section: Calendar API (calendar)">edit</a>]</span> <span class="mw-headline" id="Calendar_API_.28calendar.29"> Calendar API (calendar) </span></h3>
<p>The <a href="/dev/Calendar_API" title="Calendar API">Calendar API</a> allows you to add and modify events in the calendar for user, groups, courses, or the whole site.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=15" title="Edit section: Comment API (comment)">edit</a>]</span> <span class="mw-headline" id="Comment_API_.28comment.29"> Comment API (comment) </span></h3>
<p>The <a href="/dev/Comment_API" title="Comment API">Comment API</a> allows you to save and retrieve user comments, so that you can easily add commenting to any of your code.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=16" title="Edit section: Data definition API (ddl)">edit</a>]</span> <span class="mw-headline" id="Data_definition_API_.28ddl.29"> Data definition API (ddl) </span></h3>
<p>The <a href="/dev/Data_definition_API" title="Data definition API">Data definition API</a> is what you use to create, change and delete tables and fields in the database during upgrades.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=17" title="Edit section: Events API (event)">edit</a>]</span> <span class="mw-headline" id="Events_API_.28event.29"> Events API (event) </span></h3>
<p>The <a href="/dev/Events_API" title="Events API">Events API</a> allows to define "events" with payload data to be fired whenever you like, and it also allows you to define handlers to react to these events when they happen.  This is the recommended form of inter-plugin communication.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=18" title="Edit section: External functions API (external)">edit</a>]</span> <span class="mw-headline" id="External_functions_API_.28external.29"> External functions API (external) </span></h3>
<p>The <a href="/dev/External_functions_API" title="External functions API">External functions API</a> allows you to create fully parametrised methods that can be accessed by external programs (such as <a href="/dev/Web_services" title="Web services">Web services</a>).
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=19" title="Edit section: Message API (message)">edit</a>]</span> <span class="mw-headline" id="Message_API_.28message.29"> Message API (message) </span></h3>
<p>The <a href="/dev/Message_API" title="Message API">Message API</a> lets you post messages to users.  They decide how they want to receive them.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=20" title="Edit section: Preference API (preference)">edit</a>]</span> <span class="mw-headline" id="Preference_API_.28preference.29"> Preference API (preference) </span></h3>
<p>The <a href="/dev/Preference_API" title="Preference API">Preference API</a> is a simple way to store and retrieve preferences for individual users.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=21" title="Edit section: Portfolio API (portfolio)">edit</a>]</span> <span class="mw-headline" id="Portfolio_API_.28portfolio.29"> Portfolio API (portfolio) </span></h3>
<p>The <a href="/dev/Portfolio_API" title="Portfolio API">Portfolio API</a> allows you to add portfolio interfaces on your pages and allows users to package up data to send to their portfolios.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=22" title="Edit section: Rating API (rating)">edit</a>]</span> <span class="mw-headline" id="Rating_API_.28rating.29"> Rating API (rating) </span></h3>
<p>The <a href="/dev/Rating_API" title="Rating API">Rating API</a> lets you create AJAX rating interfaces so that users can rate items in your plugin.  In an activity module, you may choose to aggregate these to form grades.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=23" title="Edit section: RSS API (rss)">edit</a>]</span> <span class="mw-headline" id="RSS_API_.28rss.29"> RSS API (rss) </span></h3>
<p>The <a href="/dev/RSS_API" title="RSS API">RSS API</a> allows you to create secure RSS feeds of data in your module. 
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=24" title="Edit section: Tag API (tag)">edit</a>]</span> <span class="mw-headline" id="Tag_API_.28tag.29"> Tag API (tag) </span></h3>
<p>The <a href="/dev/Tag_API" title="Tag API">Tag API</a> allows you to store tags (and a tag cloud) to items in your module.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=25" title="Edit section: Time API (time)">edit</a>]</span> <span class="mw-headline" id="Time_API_.28time.29"> Time API (time) </span></h3>
<p>The <a href="/dev/Time_API" title="Time API">Time API</a> takes care of translating and displaying times between users in the site.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=26" title="Edit section: Unit test API (test)">edit</a>]</span> <span class="mw-headline" id="Unit_test_API_.28test.29"> Unit test API (test) </span></h3>
<p>The <a href="/dev/Unit_test_API" title="Unit test API">Unit test API</a> is based on Simpletest (shortly we are switching to PHP unit).  Ideally all new code should have unit tests written FIRST.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=27" title="Edit section: Web services API (webservice)">edit</a>]</span> <span class="mw-headline" id="Web_services_API_.28webservice.29"> Web services API (webservice) </span></h3>
<p>The <a href="/dev/Web_services_API" title="Web services API">Web services API</a> allows you to expose particular functions (usually external functions) as web services.
</p><p><br />
</p>
<h2><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=28" title="Edit section: Activity module APIs">edit</a>]</span> <span class="mw-headline" id="Activity_module_APIs"> Activity module APIs </span></h2>
<p>Activity modules are the most important plugin in Moodle.  There are several core APIs that service only Activity modules.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=29" title="Edit section: Activity completion API (activity completion)">edit</a>]</span> <span class="mw-headline" id="Activity_completion_API_.28activity_completion.29"> Activity completion API (activity_completion) </span></h3>
<p>The <a href="/dev/Activity_completion_API" title="Activity completion API">Activity completion API</a> is to indicate to the system how activities are completed.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=30" title="Edit section: Advanced grading API (grading)">edit</a>]</span> <span class="mw-headline" id="Advanced_grading_API_.28grading.29"> Advanced grading API (grading) </span></h3>
<p>The <a href="/dev/Advanced_grading_API" title="Advanced grading API">Advanced grading API</a> allows you to add more advanced grading interfaces (such as rubrics) that can produce simple grades for the gradebook.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=31" title="Edit section: Conditional activities API (conditional)">edit</a>]</span> <span class="mw-headline" id="Conditional_activities_API_.28conditional.29"> Conditional activities API (conditional) </span></h3>
<p>The <a href="/dev/Conditional_activities_API" title="Conditional activities API">Conditional activities API</a> is so you can tell the system how to show or hide the activity automatically. 
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=32" title="Edit section: Groups API (group)">edit</a>]</span> <span class="mw-headline" id="Groups_API_.28group.29"> Groups API (group) </span></h3>
<p>The <a href="/dev/Groups_API" title="Groups API">Groups API</a> allows you to check the current activity group mode and set the current group.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=33" title="Edit section: Gradebook API (grade)">edit</a>]</span> <span class="mw-headline" id="Gradebook_API_.28grade.29"> Gradebook API (grade) </span></h3>
<p>The <a href="/dev/Gradebook_API" title="Gradebook API">Gradebook API</a> allows you to read and write from the gradebook.  It also allows you to provide an interface for detailed grading information.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=34" title="Edit section: Plagiarism API (plagiarism)">edit</a>]</span> <span class="mw-headline" id="Plagiarism_API_.28plagiarism.29"> Plagiarism API (plagiarism) </span></h3>
<p>The <a href="/dev/Plagiarism_API" title="Plagiarism API">Plagiarism API</a> allows your activity module to send files and data to external services to have them checked for plagiarism.
</p>
<h3><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=35" title="Edit section: Question API">edit</a>]</span> <span class="mw-headline" id="Question_API"> Question API </span></h3>
<p>The question API, which can be divided into the Question bank API and the Question engine API, can be used by activities that want to use questions from the question bank in activities.
</p><p>At the moment, the Quiz module is the only example of a standard module that does this. (People have been saying wince Moodle 1.6 that the lesson module should be converted to use the question bank!) Also, the documentation is currently in a bit of a mess, but see:
</p>
<ul><li> <a href="/dev/Question_Engine_2" title="Question Engine 2">Question Engine 2</a>, in particular <a href="/dev/Using_the_question_engine_from_module" title="Using the question engine from module">Using the question engine from module</a>. That is reasonably up-to-date.
</li><li> <a href="/dev/Question_engine" title="Question engine">Question engine</a> is the old documentation that applies to Moodle 1.9 and before. The information there about the question bank half of the question system is still relevant.
</li></ul>
<h2><span class="editsection">[<a href="/dev/index.php?title=Core_APIs&amp;action=edit&amp;section=36" title="Edit section: See also">edit</a>]</span> <span class="mw-headline" id="See_also"> See also </span></h2>
<ul><li> <a href="/dev/Plugins" title="Plugins">Plugins</a> - plugin types also have their own APIs
</li></ul>
